EmailJS Setup Instructions for AstraLock

This guide will help you set up EmailJS to enable real email verification for your AstraLock extension.

Note: The current implementation includes a demo mode that shows the verification code directly in the UI. This is for development purposes only and should be replaced with real email sending in production.

Step 1: Create an EmailJS Account

Visit https://www.emailjs.com/ and sign up for a free account.

The free tier allows 200 emails per month, which should be sufficient for testing and personal use.

Step 2: Add an Email Service

After signing in:

  1. Go to "Email Services" in the dashboard
  2. Click "Add New Service"
  3. Choose your preferred email provider (Gmail, Outlook, etc.)
  4. Follow the instructions to connect your email account
  5. Give your service a name (e.g., "AstraLock")

Once created, note down the Service ID (e.g., "service_abc123").

Step 3: Create an Email Template

Next, create a template for your verification emails:

  1. Go to "Email Templates" in the dashboard
  2. Click "Create New Template"
  3. Design your email template with the following variables:
    • {{to_name}} - The recipient's name
    • {{verification_code}} - The verification code
  4. Save your template

Here's a simple template example:

Subject: Your AstraLock Verification Code

Hello {{to_name}},

Your verification code for AstraLock is: {{verification_code}}

This code will expire in 15 minutes.

If you did not request this code, please ignore this email.

Best regards,
AstraLock Team

Once created, note down the Template ID (e.g., "template_xyz789").

Step 4: Get Your Public Key

To authenticate with EmailJS:

  1. Go to "Account" in the dashboard
  2. Find your Public Key under "API Keys"

Note down this key (e.g., "user_abcdefg123456").

Step 5: Update Your Code

Now, update your AstraLock code with the values you've collected:

  1. Open popup.js and content.js
  2. Find the commented EmailJS sections
  3. Uncomment the EmailJS code
  4. Replace the placeholder values:
    • YOUR_EMAILJS_PUBLIC_KEY with your Public Key
    • YOUR_SERVICE_ID with your Service ID
    • YOUR_TEMPLATE_ID with your Template ID
  5. Comment out or remove the demo code that displays the verification code in the UI

Testing Your Implementation

After updating your code:

  1. Load your extension in Chrome
  2. Try the "Forgot Password" flow
  3. Check your email for the verification code

If everything is set up correctly, you should receive an email with your verification code.

Troubleshooting: If you're not receiving emails, check the browser console for errors. Make sure your EmailJS service is connected properly and that you've entered all IDs correctly.
